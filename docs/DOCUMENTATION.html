🔍 阶段一：车牌检测 (PlateDetector)

混合策略：YOLO 优先 + 形态学兜底

为什么需要混合策略？
YOLO 模型在全车照片上表现优异（精确定位车牌），但对裁剪后的车牌图片（没有车身上下文）会失效。形态学方案恰好相反——不依赖上下文但容易误检（如把 MINI 车标当成车牌）。两者互补。

策略 A：YOLO 深度学习检测

使用 OnnxOCR 的 car_plate_detect.onnx 模型，基于 YOLOv5 架构，输入 640×640 图片，输出 25200 个候选框（含 4 角关键点 + 车牌类型分类）。

PlateDetector.cpp — YOLO 预处理
// Letterbox resize: 保持宽高比, 灰色(114)填充到 640x640 void PlateDetector::letterbox(const cv::Mat &img, cv::Mat &out, double &scale, int &padX, int &padY, int targetSize) { int h = img.rows, w = img.cols; scale = std::min((double)targetSize / h, (double)targetSize / w); int newH = (int)(h * scale), newW = (int)(w * scale); cv::Mat resized; cv::resize(img, resized, cv::Size(newW, newH)); out = cv::Mat(targetSize, targetSize, CV_8UC3, cv::Scalar(114, 114, 114)); // 灰色填充 padX = (targetSize - newW) / 2; padY = (targetSize - newH) / 2; resized.copyTo(out(cv::Rect(padX, padY, newW, newH))); }
PlateDetector.cpp — YOLO 输出解码
// 模型输出: [1, 25200, 16] // 字段: [cx, cy, w, h, obj_conf, // x1, y1, x2, y2, x3, y3, x4, y4, (4角关键点) // p_blue, p_yellow, p_green] (车牌类型概率) for (int i = 0; i < numDet; i++) { const float *det = data + i * numFields; float objConf = det[4]; if (objConf < YOLO_CONF_THRESH) continue; // 类别概率 (位置13,14,15, 已sigmoid) float pBlue = det[13], pYellow = det[14], pGreen = det[15]; float maxProb = std::max({pBlue, pYellow, pGreen}); float score = objConf * maxProb; // 坐标从640空间还原到原图 float cx = (float)((det[0] - padX) / scale); float cy = (float)((det[1] - padY) / scale); // 关键过滤: 宽高比 2.0~6.0 float aspect = (float)bw / bh; if (aspect < MIN_ASPECT || aspect > MAX_ASPECT) continue; }
宽高比过滤是关键！
YOLO 有时会把整辆车检测为一个候选（如蓝色 Mini Cooper，bbox 宽高比 ≈ 1.07）。标准中国车牌是 440mm × 140mm，宽高比 ≈ 3.14。过滤条件 2.0~6.0 可以有效排除整车的误检。

策略 B：形态学检测（兜底）

当 YOLO 无法检测时（如纯车牌裁剪图），回退到传统图像处理方案：

// 1. Sobel 边缘检测 (水平方向, 车牌字符有强垂直边缘) cv::Sobel(gray, sobelX, CV_16S, 1, 0, 3); // 2. 多尺度形态学闭运算 (合并字符区域) cv::morphologyEx(binary, morph, cv::MORPH_CLOSE, kernel); // 3. HSV 蓝色区域兜底检测 cv::inRange(hsv, cv::Scalar(95,50,40), cv::Scalar(130,255,255), blueMask); // 4. 候选验证打分: 蓝色比例 + 中心蓝色 + 边缘密度 int score = verifyCandidate(bgrRegion); if (score >= 1) results.push_back(region);

🧠 阶段二：ONNX 文字识别 (PlateOCR)

模型规格

属性值
模型文件plate_rec.onnx (649 KB)
输入Float [1, 3, 48, 168] — RGB 归一化图像
输出Float [1, 21, 78] — 21 时间步 × 78 字符类别
归一化(pixel/255 - 0.588) / 0.193
解码CTC Greedy Decode
字符集31 省份 + 10 特殊 + 10 数字 + 24 字母 + 2 其他 = 78

预处理流程

cv::Mat PlateOCR::preprocess(const cv::Mat &plateCrop) { // 1. 高质量缩放到模型输入尺寸 cv::Mat resized; cv::resize(plateCrop, resized, cv::Size(168, 48), 0, 0, cv::INTER_CUBIC); // 2. BGR → RGB (模型训练时用 RGB) cv::Mat rgb; cv::cvtColor(resized, rgb, cv::COLOR_BGR2RGB); // 3. 归一化: (x/255 - mean) / std cv::Mat floatImg; rgb.convertTo(floatImg, CV_32FC3, 1.0 / 255.0); floatImg = (floatImg - 0.588f) / 0.193f; // 4. HWC → CHW blob: [1, 3, 48, 168] return cv::dnn::blobFromImage(floatImg); }

CTC 贪心解码

模型输出 21 个时间步，每步 78 个字符的 logit。CTC 解码规则：

// 对每个时间步取 argmax int maxIdx = 0; for (int c = 1; c < numClasses; c++) if (timestep[c] > timestep[maxIdx]) maxIdx = c; // 跳过 blank (index 0) 和连续重复 if (maxIdx != 0 && maxIdx != prevIdx) { float prob = 1.0f / (1.0f + std::exp(-timestep[maxIdx])); indices.push_back(maxIdx); probs.push_back(prob); } prevIdx = maxIdx; // 映射索引到字符: QString charStr = QString::fromUtf8(PLATE_CHARS); // 例: index 10 → '苏', index 52 → 'A', index 42 → '0'
CTC 解码直觉理解：
想象模型像打字一样，从左到右逐个"看"车牌图片。每个时间步它"打"一个字符（或空白）。如果它连续"打"了两个相同的字符（比如看到'8'很宽，占了两个时间步），CTC 会把它们合并成一个。空白符（index 0）表示"这里没有字符"，直接跳过。

OCR 验证回退机制

PlateRecognizer 会遍历所有检测候选，逐一做 OCR 并验证结果合法性：

// 验证规则: 标准蓝牌 7 字符, 新能源牌 8 字符 static bool isValidPlate(const QString &plate) { if (plate.length() < 7 || plate.length() > 8) return false; // 首字符必须是省份简称 if (!provinces.contains(plate[0])) return false; // 第二位必须是字母 A-Z if (plate[1] < 'A' || plate[1] > 'Z') return false; return true; } // 逐候选遍历, 找第一个通过验证的 for (int rank = 0; rank < sortedIdx.size(); rank++) { QString plate = m_ocr.recognize(candidate.plateImage, conf); if (conf >= 0.85 && isValidPlate(plate)) { foundValid = true; break; // 找到合法结果, 停止搜索 } }

信号槽通信

核心层通过 Qt 信号通知 UI 进度：
recognitionStarted() → progressChanged(%) → recognitionFinished(result)
支持跨线程 QueuedConnection。

线程安全

RecognitionResult 通过 Q_DECLARE_METATYPE 注册，可在信号槽中跨线程传递。识别任务在后台线程执行，UI 不阻塞。
